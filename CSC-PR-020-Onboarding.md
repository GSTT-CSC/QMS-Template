# Onboarding for new starters
<!-- [13485:5.3d,6.2a,6.2b,6.2c,6.2d6.2e] -->

## General

## Purpose
This document is a guidance for new starters and their supervisor to perform an effective induction into their new role
within the Clinical Scientific Computing team. This document should be used as part of a dedicated onboarding meeting 
scheduled by the supervisor on the first day of work for the new starter.

## Scope

This process should be followed for all substantive employees employed by the CSC team. An abridged version of this 
procedure may be useful for interns, trainees, visitors. This procedure does not cover the organisational induction 
activities e.g. mandatory fire training, information governance etc. as that is covered by the Trust induction the new 
starter will have attended prior to this onboarding.


## Roles and Responsibilities

| Role         | Responsibility                                                                                                               |
|--------------|------------------------------------------------------------------------------------------------------------------------------|
| Line Manager | To onboard the new start using this procedure and to ensure that any subsequent actions are completed in an appropriate time |
| New starter  | To attend the onboarding meeting, make appropriate notes and implement any actions that result from the meeting              |                                                                                                                                                                                           

## Procedure

### 1. Before first day
- Order laptop

### 2. First day

-	Building access
    -	Local Office

-	Tour
    -	Water, toilets etc.
    -	Department/People

-	Clinical Scientific Computing introduction
    -	History
    -	Strategy
    -	Roadmap

-	Expectations
    -	Job Description
    -	Admin
        -	Annual leave
-	Chat system
    -	Microsoft Teams
        -	Standup minutes rota (if core team member)
    -	Apply standard email signature 

-	Project management system
    -	Share calendar
    -	Shared drive

-	Send invites to services and systems
    -	GitHub
    -	XNAT

-	Introduction to projects
    - Local Projects


-	Arrange 1-on-1s
    -	Arrange intros
    -	Regular with line manager
    -	3-month PDR setting
    -	Invite to regular meetings
        -	Stand-ups
        -	Workshops
        -	Project specific meetings

### 3. First week/month
(Depending on delivery of laptop)
- Edit the About Page
- QMS induction
    - Quality manual
    - Training log
    - Clinical risk management training (Stage 1)
- Complete Trust mandatory training
- Reading list assignment
- Fix documentation on any appropriate project
- Fix a bug on any appropriate project
- Improve test coverage on any appropriate project
- Implement feature on any appropriate project
